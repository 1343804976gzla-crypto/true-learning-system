# 📊 数据看板功能说明

## 功能概述

数据看板提供错题消耗进度的实时量化分析，帮助你清晰了解：
- 今日学习成果
- 错题积压情况
- 预期清仓时间
- 学习趋势变化

## 📈 核心指标说明

### 1. 今日消除量
- **定义**: 今天（00:00-23:59）状态变更为"已归档"的错题数量
- **意义**: 衡量今日学习成果
- **子指标**: 今日重做量（不论对错）

### 2. 7天日均新增
- **定义**: 过去7天新创建的错题总数 ÷ 7
- **意义**: 了解错题产生速度
- **用途**: 判断学习难度是否合理

### 3. 当前错题积压量
- **定义**: 尚未归档的活跃状态错题总数
- **意义**: 当前需要消化的错题总量
- **目标**: 逐步降低至 0

### 4. 预计清仓天数
- **公式**: `当前错题积压量 ÷ (每日计划复习量 - 7天日均新增)`
- **意义**: 按当前速度多久能清空积压
- **特殊情况**:
  - 显示 `∞`: 新增速度 ≥ 复习速度，无法清仓
  - 显示 `0`: 已清仓，无积压

### 5. 每日需做错题
- **定义**: 为达到清仓目标，每天需要做的错题数量
- **计算逻辑**:
  - 可清仓时：等于每日计划复习量
  - 无法清仓时：建议值 = 7天日均新增 + 5

### 6. 净日均进度
- **公式**: `每日计划复习量 - 7天日均新增`
- **意义**: 实际每天能消化的错题数量
- **目标**: 保持正值且尽可能大

## 🎯 使用建议

### 1. 设定合理的每日计划

- **新手**: 10-15 题/天
- **进阶**: 20-30 题/天
- **冲刺**: 40-50 题/天

### 2. 关注净日均进度

- **理想状态**: 净进度 > 10 题/天
- **警戒状态**: 净进度 < 5 题/天
- **危险状态**: 净进度 ≤ 0（无法清仓）

### 3. 优先处理高严重度错题

- 🚨 **致命盲区 (Critical)**: 自信但答错，优先级最高
- 🚑 **顽固病灶 (Stubborn)**: 同题错≥2次，需要强化
- ⚠️ **隐形地雷 (Landmine)**: 不确定但答对，需要巩固
- 📋 **普通错题 (Normal)**: 正常复习即可

### 4. 定期查看本周趋势

- 观察新增和消除的平衡关系
- 如果连续多天净变化为正（+），需要提高复习量
- 如果连续多天净变化为负（-），说明进度良好

## 🔧 技术实现

### 后端

- **文件**: `routers/dashboard.py`
- **框架**: FastAPI + SQLAlchemy
- **数据库**: SQLite
- **模型**: `WrongAnswerV2`, `WrongAnswerRetry`

### 前端

- **文件**: `templates/dashboard_stats.html`
- **样式**: Neumorphism CSS
- **交互**: 原生 JavaScript + Fetch API

### 数据流

```
用户访问页面
    ↓
前端加载 → 调用 API (/api/dashboard/stats?daily_planned_review=20)
    ↓
后端实时查询数据库
    ↓
计算 5 个核心指标
    ↓
返回 JSON 数据
    ↓
前端渲染 Neumorphism 卡片
```

## 📝 数据库字段说明

### WrongAnswerV2 表

关键字段：
- `mastery_status`: 状态（active/archived）
- `archived_at`: 归档时间
- `created_at`: 创建时间
- `severity_tag`: 严重度标签（critical/stubborn/landmine/normal）

### WrongAnswerRetry 表

关键字段：
- `wrong_answer_id`: 关联的错题 ID
- `retried_at`: 重做时间
- `is_correct`: 是否正确

## 🐛 故障排查

### 问题 1: 页面显示 "∞" 天

**原因**: 新增速度 ≥ 复习速度，无法清仓

**解决方案**:
1. 提高每日计划复习量
2. 降低学习难度，减少新增错题
3. 参考"每日需做错题"建议值

### 问题 2: 数据不更新

**原因**: 浏览器缓存

**解决方案**:
1. 点击"🔄 刷新数据"按钮
2. 清除浏览器缓存（Ctrl+Shift+Delete）
3. 硬刷新页面（Ctrl+F5）

### 问题 3: API 返回 500 错误

**原因**: 数据库查询失败

**解决方案**:
1. 检查数据库文件是否存在
2. 查看服务器日志
3. 运行 `python test_dashboard_api.py` 诊断
